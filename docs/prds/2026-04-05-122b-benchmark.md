# PRD: Benchmark Qwen3.5-122B-A10B on M5 Max 128GB

**Version:** 1.0
**Date:** 2026-04-05
**Bead:** sylveste-e25

## Problem

The C3 tier in interfer's routing table is unvalidated. We don't know if
Qwen3.5-122B-A10B runs on M5 Max 128GB, how fast it is, or whether its
quality justifies the memory cost over the 35B or the latency cost over
the 397B via flash-moe.

## Deliverables

1. **Benchmark results TSV** — tok/s, prompt tok/s, TTFT, peak memory for
   122B (MLX) and 397B (flash-moe) across 4+ test prompts
2. **Quality comparison** — side-by-side outputs on coding tasks
3. **Routing recommendation** — which model for C3 tier, with data

## Success Metrics

| Metric | Target |
|--------|--------|
| 122B loads without OOM | Yes/No |
| 122B generation speed | Measure (expect 15-40 tok/s) |
| Quality comparison | 122B vs 397B on 4 coding tasks |
| Routing recommendation | Data-backed C3 tier assignment |

## Non-Goals

- Not optimizing the 122B (KV quantization, early exit — future work)
- Not benchmarking the 35B (already done: 86 tok/s)
- Not running flash-moe throughput benchmarks (already done: 1.98 tok/s)
